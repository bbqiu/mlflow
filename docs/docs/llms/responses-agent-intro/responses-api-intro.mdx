# Responses API Introduction

The Responses API, introduced by OpenAI, is a superset of features that the Chat Completions API had (largely shared by <APILink fn="mlflow.pyfunc.ChatModel">`ChatModel`</APILink> and <APILink fn="mlflow.pyfunc.ChatAgent">`ChatAgent`</APILink>). See their documentation for more details:

- [Responses vs Chat Completions](https://platform.openai.com/docs/guides/responses-vs-chat-completions)
- [Responses API Docs](https://platform.openai.com/docs/api-reference/responses)

Below are some of the differences in representation for common use cases between the two APIs.

### Standard Text Response

#### Responses API

```json
{
  "type": "message",
  "id": "",
  "content": [{ "annotations": [], "text": "", "type": "output_text" }],
  "role": "assistant",
  "status": "completed"
}
```

#### ChatCompletions API

```json
{
  "role": "assistant",
  "content": ""
}
```

---

### Tool Calls

#### Responses API

```json
{
  "type": "function_call",
  "id": "fc_1",
  "arguments": "",
  "call_id": "call_1",
  "name": "",
  "status": "completed"
}
```

#### ChatCompletions API

```json
{
  "role": "assistant",
  "content": "",
  "tool_calls": [
    {
      "id": "call_1",
      "type": "function",
      "function": { "name": "", "arguments": "" }
    }
  ]
}
```

---

### Tool Call Results

#### Responses API

```json
{
  "type": "function_call_output",
  "call_id": "call_1",
  "output": ""
}
```

#### ChatCompletions API

```json
{
  "role": "tool",
  "content": "12",
  "tool_call_id": "call_1"
}
```

---

### Tool Definitions

#### Responses API

```json
{
  "name": "",
  "parameters": {},
  "strict": true,
  "type": "function",
  "description": ""
}
```

#### ChatCompletions API

```json
{
  "type": "function",
  "function": {
    "name": "",
    "description": "",
    "parameters": {},
    "strict": true
  }
}
```
